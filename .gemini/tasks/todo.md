# Wedding Quest V2 現代化改版計畫

### [階段一] 任務目標與範圍
本階段目標為根據 `devSpec_v2.md` 將目前的數位婚禮邀請卡升級為「現代化 J-RPG 風格」。
範圍涵蓋：
1. 視覺多字體混搭、玻璃擬物化對話框、以及像素藝術的最佳化。
2. 更新回憶劇本，並強化全螢幕熱區與平滑文字轉場。
3. 新增白光閃爍轉場特效，無縫切換至教堂背景。
4. 最終需透過 Browser Subagent 執行自動化測試並交付 CI/CD。

### [執行步驟] 詳細執行步驟
- [ ] 1. **重塑視覺風格 (`src/style.css`)**
   - 實作玻璃擬物化 (`rgba(15,23,42,0.7)` + `backdrop-filter: blur(8px)`)。
   - 套用字體混搭，內文使用無襯線字體，並調整行高為 1.6。
   - 確保所有像素圖片包含 `image-rendering: pixelated;`。
   - 加入 `@keyframes` 動畫（呼吸動畫、點擊繼續的浮動動畫、全螢幕白光閃爍）。
- [ ] 2. **調整遊戲互動與邏輯 (`src/main.ts`)**
   - 確保 `.dialogue-wrapper` 覆蓋全螢幕，點擊即可切換句子或加速打字。
   - 對應新劇本修改說話者高亮邏輯（社長 vs 學姐）。
   - 在關鍵對話（登入婚姻伺服器）觸發全螢幕閃白過渡效果，並切換背景與 BGM。
- [ ] 3. **更新對話資料 (`src/data/script.json`)**
   - 替換為規格書提供的 11 句新劇本陣列。
- [ ] 4. **自動化測試與 CI/CD 送交**
   - 透過 Subagent 以瀏覽器開啟本地開發伺服器進行測試。
   - 測試確認無誤後，透過 Git 建立 commit 並推播。

### [預期結果] 驗證標準
1. 視覺層次表現符合預期（玻璃質感、字體清晰、像素銳利）。
2. 發言角色切換活躍度，且白光轉場能順利觸發。
3. Browser Subagent 自動操作可跑完所有對話無 JS Error。
4. 完成版本控制紀錄與發布。
